# home-manager — Context for AI Agent

`~/.dotfiles/home-manager.configsymlink/`. Activated via `home-manager switch --impure --flake ~/.dotfiles/home-manager.configsymlink` (the `--impure` is required — see "Private dotfiles" below).

## Files

- **`flake.nix`** — inputs: `nixpkgs`, `home-manager`, `nixGL` (no `private-dotfiles` or `work-dotfiles` flake inputs — see resolution below). `mkHost` builds a `homeManagerConfiguration` from shared modules + extra modules. `nixFilesFrom` auto-loads root `.nix` files from both `private-dotfiles/` and `work-dotfiles/` as modules. `hosts/*.nix` in private-dotfiles return `homeConfigurations` attrset fragments keyed by `username@static-hostname`. Bare-username aliases for auto-detection live in `bare-aliases.nix`. `nix.conf` `warn-dirty = false` is required for home-manager auto-detection to work with jj (dirty-tree warnings on stdout would otherwise pollute `nix eval`). `hm-session-vars.sh` is sourced from `zsh/zshenv.symlink` (zsh is not managed by home-manager).
- **`bare-aliases.nix`** — at eval time, reads `/etc/hostname` (FQDN and short form both tried) and exposes any `user@<live-host>` entry under bare `user`, so `home-manager switch --impure --flake ~/.dotfiles/home-manager.configsymlink` (no explicit `#user@host`) auto-detects via `$USER`. Pure-eval-safe: empty `/etc/hostname` produces no aliases. **Caveat**: the flake snapshot is read from git's index, so any new file under `home-manager.configsymlink/` (or new `hosts/<hostname>.nix`) needs `git add` before the next switch — jj's auto-snapshot does not bridge to the git index for `git+file://` flake inputs.
- **`home.nix`** — packages, unfree predicate (albert), copyq daemon, declares periodic jobs via `dotfiles.schedule.jobs` (sync-repos, updatedb, battery-notify, flake-update — see `schedule.nix`). Defines small inline derivations consumed by the OSDs: `osd` (local Python package built from `xwindow/osd/`, used by battery-osd + hangul-osd) and `jejuhallasan-ttf` (a `fetchurl` of one ttf from `google/fonts` SIL OFL 1.1 — preferred over `pkgs.google-fonts` which is 2.3 GB). The `hangul-osd` `writeShellScriptBin` wrapper exports `GI_TYPELIB_PATH` (Pango/PangoCairo/cairo/IBus/harfbuzz typelibs from nix store + gobject-introspection wrapper for cairo's `cairo-1.0.typelib`, which Pango pulls in transitively) and `HANGUL_OSD_FONT_FILE` (path to the ttf above) before exec'ing the inner `writePython3Bin` impl. Both env vars are required because hangul-osd uses Pango (typelibs must be resolvable) and registers JejuHallasan as a fontconfig app-font (`HANGUL_OSD_FONT_FILE` is read at startup) — see `xwindow/AGENTS.md` for why.
- **`gnome.nix`** — dconf settings (key repeat, mouse speed, cursor size 64, Korean input Sebeolsik 390, disable gnome-panel/desktop), IBus integration (`pkgs.ibus` for the GTK4 IM module, `~/.config/environment.d/30-ibus.conf` for `GTK_IM_MODULE` / `GTK_PATH` / etc.), gnome-flashback systemd drop-ins (xmonad session requires `gnome-flashback.target` + service-restart override). Declares `random-lockscreen` via `dotfiles.schedule.jobs` and `hangul-osd` as a `systemd.user.services.*` (`PartOf=graphical-session.target`) — gnome-only because both depend on a graphical session and ibus.
- **`schedule.nix`** — cross-backend scheduler. `dotfiles.schedule.backend = "systemd"` (default) emits `systemd.user.{services,timers}`; `"cron"` emits a managed crontab block via `install-crontab.sh` activation. One job declaration → one of two backends, picked per host. Per-job options: `enable` (default true; set false on hosts where the job doesn't apply, e.g. battery-notify on a battery-less host), `description`, `command` (with `%h` expansion for cron), `schedule.{systemd,cron}` (both required — no auto-translation), `nice`, `ioSchedulingClass` (mapped to `ionice -c` under cron), `randomizedDelaySec` (native under systemd; emitted as `sleep $((RANDOM \% N))` under cron, with `\%` cron-escaped to bash `%`), `persistent` (native under systemd; no-op under cron), `env` (per-job; `Environment=` under systemd, `/usr/bin/env KEY=VAL` prefix under cron). Top-level `dotfiles.schedule.environment` and `pathExtra` only apply to cron mode (default `pathExtra` covers `~/.nix-profile/bin`, `~/.local/bin`, `/run/current-system/sw/bin`, `/usr/local/bin`); systemd mode picks env from `~/.config/environment.d/` (managed by `private-dotfiles/logger.nix`, which mirrors `LOG_ROOT`/`LOG_REL_BASE` into `dotfiles.schedule.environment` so cron jobs see the same env). **Why this exists**: nix systemd ≥256 hard-requires cgroup v2 — `systemd --user` exits silently with rc=1 after `statfs("/sys/fs/cgroup/")` returns `TMPFS_MAGIC` instead of `CGROUP2_SUPER_MAGIC`. Switching the host to cgroup v2 needs a kernel cmdline change (`systemd.unified_cgroup_hierarchy=1`) and a reboot, neither of which is feasible on every host (e.g. managed images that you can't reflash). cron is universal and good enough for the scheduling features we actually use.
- **`install-crontab.sh`** — activation helper for the cron backend. Reads existing crontab (`crontab -l`), strips any prior `# >>> home-manager managed (dotfiles.schedule) >>>` … `# <<< home-manager managed (dotfiles.schedule) <<<` block via awk, appends the new block, installs via `crontab -`. Marker contract preserves user-edited entries outside the block. Idempotent: `cmp -s` skips reinstalling identical content. Activation prepends `/usr/bin:/usr/sbin` to PATH so the system `crontab(1)` is found from home-manager's minimal-PATH activation env.
- **`neovide.nix`** — nixGL-wrapped neovide (GPU access on non-NixOS), font-copying activation (JetBrains Mono + Nerd Font + Source Code Pro for neovide's default fallback).
- **`nvim.nix`** — neovim (default editor, vi/vim aliases). `initLua` sources `myinit.lua` (nix generates `init.lua` which overwrites `nvim.configsymlink/init.lua` on every switch — `init.lua` is gitignored). Dev tool packages (LSPs, linters, formatters, DAP deps), rustaceanvim (Rust LSP). Coverage table documents all tools per language. See `nvim.configsymlink/AGENTS.md`.
- **`xmonad.nix`** — xmonad + contrib via nix 0.18, xfce4-panel + xfconf, xfconf dbus activation hook. See `xwindow/AGENTS.md`.
- **`xdg.nix`** — firefox-container desktop entry + mimeapps.
- **`zmx.nix`** — prebuilt zmx binary from `zmx.sh` (session persistence tool; source build via zmx's flake fails because `zig2nix` cannot vendor ghostty's `git+https?ref=HEAD` dependency).
- **`system-deps.sh`** — **bootstrap-only** (run from `script/install`, NOT `home-manager switch`). Detects package manager (dnf/yum preferred for Amazon Linux); installs GNOME packages + xmonad session files when `gnome-session` is present; copies keyd configs to `/etc/keyd` and enables the service when `/dev/input` exists; runs `loginctl enable-linger`. Also idempotently patches `/usr/share/X11/xkb/symbols/inet` so keycodes 198/202 (`KEY_F20`/`KEY_F24`) map to `F20`/`F24` keysyms — keyd's Super+C/V macro emits these for neovide, and `setxkbmap` (called by ibus engine switches) would otherwise wipe an `xmodmap` override (see `keyd/AGENTS.md`). **Re-run `script/install` after `apt upgrade xkeyboard-config`** since the package may overwrite the file (backup at `inet.dotfiles-bak`).

## Private dotfiles resolution

`private-dotfiles` is gitignored and resolved at flake eval time via `builtins.getEnv "HOME"` — that's why `--impure` is required. A sentinel path falls back to empty when unavailable, so pure eval still succeeds. `work-dotfiles` follows the same pattern. See each sibling repo's own `AGENTS.md` for layout.

## Known issues / Constraints

- **Nix flakes and gitignored files**: flakes copy git-tracked source tree to store. Two ways to include gitignored content: (1) keep it in a separate colocated repo and use `git+file://` (no `--impure` needed but `flake.lock` then bakes a per-machine path — bad for portability), or (2) resolve the path from `$HOME` via `builtins.getEnv` at eval time and run `--impure` (portable `flake.lock` — this is what private-dotfiles uses).
- **`warn-dirty`**: jj always has uncommitted changes; `nix eval` outputs the warning to stdout, breaking home-manager's string comparison. Fix: `warn-dirty = false` in `~/.config/nix/nix.conf`.
- **xfconf needs dbus service registration** — handled by Home Manager activation.
- **Fonts need copying to `~/.local/share/fonts`** for neovide / dzen2 (nix font paths aren't read by skia / dzen2).
- **Nix-installed GTK apps don't show in xfce4-panel systray** (library mismatch).
